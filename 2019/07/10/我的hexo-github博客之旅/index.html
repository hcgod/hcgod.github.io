小白的hexo+github博客之旅

Veröffentlicht am 2019-07-10

起因

最近一直在学习关于如何做海外的B2B去平台化品牌营销。 后来发现现在网上的信息都太少了，更多的是关于电商B2C的，所以自己决定写个博客，把自己的这些实操经验都记录下来，就想到了博客。

目的

– 第一，记录自己的操作经验
– 第二，找回写作的习惯（PS: 曾经自己是有习惯写作的，那个时候写空间，都是关于自己的。）
– 第三，为B2B去平台化的海外营销做个操作范例。

纠结

好吧，这就开动吧。想到要写博客，首先想到的是wordpress，这些年自己操作了不少的建站都是基于WordPress写的，相对比较熟悉。然后，我需要一个VPS，一个域名，后来左思右想不舍得去投入VPS就放弃了。便宜的稳定性差，稳定性好的价格太贵。加上以前很多次失败的教训，告诉我再没有绝对的把握之前就采购这些资产显然是不明智的。
后来在网上看到有人在介绍hexo+github的方法，本不想跟进的，但是实在没法找到一个方案可以解决我的问题。就抱着试一试的心态开始了hexo+github的建站之路。

开动

首先，感谢[吴润]的知乎文章，GitHub+Hexo 搭建个人网站详细教程。
其次，本次搭建过程中也遇到一些问题在以上文章中并没有提到，这里就说说那些对我这个小白造成阻碍的问题。

  • 文中提到： Hexo就是我们的个人博客网站的框架， 这里需要自己在电脑常里创建一个文件夹，可以命名为Blog, 如果你这里建立了blog文件夹后安装hexo，然后在继续在该文件夹出初始化blog

，会在该文件夹继续生产一个 blog文件夹。后续的hexo命令操作一定要到新的文件夹里面执行。
也就是，之前是比如我的，在初始化前是C:\Users\RYZEN\blog， hexo程序初始化后就要到新的blog文件夹执行hexo命令。例如： C:\Users\RYZEN\blog\blog>hexo s

  • 文中提到: GitHub创建个人仓库，仓库名要用固定写法:用户名.github.io, 这里补充一点， 一个github账户，只能对应一个hexo网站，如果想要通过新建仓库的方式来新建网站是行不通的。

这样，网站就建好了。整体上没什么大的问题。

但是，建好后，我是非常不喜欢这个landscape的主题。于是就换个主题呗。 next，最好的选择。

新的问题出现BOOM!
git clone的速度太慢，尝试了汪小九的proxy修改方案，不行， 要报错， Proxy CONNECT aborted. 找了一堆解决方案，都不行。
最后还是决绝了
用的是知乎网友风木草的 方法，该评论就在汪小九方法的下面。

就这样，完美收工。
